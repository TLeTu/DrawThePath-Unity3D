# Prompt to Build Python Procedural Level Generator

You are an expert Python game developer.  
Your task is to write a **Python script** that generates procedural levels for a tile-based game, based on a set of input parameters.  

## Requirements

**Language & Output**
- The script must be written in **Python 3**.
- The generated level must be exported to a JSON file following the format described below.

**Input Parameters**
The script must accept the following parameters:
1. `map_width`: integer, width of the grid.
2. `map_height`: integer, height of the grid.
3. `path_width`: integer, how many tiles wide the main path should be (minimum = 1).
4. `num_enemies`: integer, how many enemies to place.
5. `num_path_turns`: integer, how many turns the main path from start to end should have (to avoid straight lines).

**Rules for Level Generation**
1. The grid is represented as `width x height`.
2. Each cell is either:
   - `0` → unwalkable  
   - `1` → walkable
3. The **main path**:
   - Connects a **start tile** (on one edge of the map) to an **end tile** (on the opposite edge).
   - Has `path_width` tiles in thickness.
   - Includes at least `num_path_turns` turns so it’s not too linear.
   - May include **alternative branches** or **open areas** (e.g., 2×2 or 3×3 walkable regions).
4. **Enemies**:
   - Each enemy moves between two coordinates (`startX`, `startY`, `endX`, `endY`).
   - Enemies must **not spawn directly on the main path**.
   - Their patrol path must be walkable (`1`s) and must intersect or cross the main path at least once, ensuring the player encounters them.
   - Enemy patrol routes should be randomized within the constraints.
5. The start and end tile coordinates must be saved (`startTileX`, `startTileY`, `endTileX`, `endTileY`).

**Output JSON Format**
The output file must have this structure:
```json
{
  "width": <map_width>,
  "height": <map_height>,
  "tiles": [
    { "row": [0, 1, 1, ...] },
    { "row": [1, 0, 0, ...] }
  ],
  "startTileX": <int>,
  "startTileY": <int>,
  "endTileX": <int>,
  "endTileY": <int>,
  "enemies": [
    { "startX": <int>, "startY": <int>, "endX": <int>, "endY": <int> }
  ]
}
```

**Additional Notes**
- The level should always guarantee **a valid walkable path** from start to end.
- Randomness is encouraged, but respect the constraints (path width, turns, enemy placement).
- Use Python’s `random` module for variability.
- Ensure the script is modular (functions for path generation, enemy placement, JSON export).
- Save the result to `generated_level.json`.
